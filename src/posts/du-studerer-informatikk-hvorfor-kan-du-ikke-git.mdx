---
title: Du studerer informatikk. Hvorfor kan du ikke Git?
date: 2025-03-13
description: Vi lærer nevrale nettverk og OS-programmering, men mange IFI-studenter kan ikke Git. Det må vi gjøre noe med.
---

> Vi lærer å programmere, trene nevrale netverk, og bygge OS. Likevel står mange IFI-studenter fortsatt og fomler med helt grunnleggende versjonskontroll. Å gå på IFI uten å kunne git er som å gå på barneskolen uten å kunne lese. Vi må slutte å late som Git er valgfritt.
> 

Git er industristandard. Fra små hobbyprosjekter til globale teknologiselskaper er Git selve navet i utviklingsarbeidet. Tenk på Git som tidsreise: Du kan gå tilbake i historien, lage nye tidslinjer (branches), teste ideer, og flette dem sammen igjen. Det handler ikke bare om å lagre kode, men om struktur, samarbeid og sporbarhet. Git er ikke bare «litt praktisk». Det er fundamentalt.

De fleste av oss møter Git for første gang i et gruppeprosjekt. Under tidspress. Sammen med tre andre som heller ikke helt kan det. Det er skummelt når hele gruppa hviler på at én person “kan litt Git”. Én feil merge, én force push, og plutselig sitter alle og stirrer på en ødelagt historikk. Git burde ikke være noe man lærer når hele gruppa står og holder pusten. Du må lære det før det gjelder, når du bare kan ødelegge for deg selv. Ja, Git kan være tuklete. Ja, setup kan være en terskel. Merge conflicts føles unødvendig komplisert. Nettopp derfor må vi starte tidlig, så det får satt seg før det blir kritisk.

Du kan ikke sitte og sutre over at jobbmarkedet er vanskelig hvis du ikke gidder å lære deg verktøyene arbeidsgivere forventer at du kan. Commits, branching og pull requests er ikke ekstra innsats. Det er minimum. Jeg har vært i jobbintervjuer der trygghet i grunnleggende Git faktisk var nok til å skille meg ut. Ikke noe avansert, bare solid forståelse av hvordan man jobber strukturert i team. Hvis vi vil at IFI-studenter skal være mer attraktive enn NTNU-plebs på jobbmarkedet, må vi slutte å undervurdere de praktiske ferdighetene. 

Her er hva vi faktisk kan gjøre. Git må inn som en obligatorisk del av IN1010 og IN1030, med øving og konkret bruk, ikke bare en rask gjennomgang. Flere emner bør bruke Git aktivt til distribusjon av oppgaver og ressurser, slik at studentene jevnlig må clone, committe og pushe. Alle studenter burde lage ett eget repo hvert semester for notater, småprosjekter og øving, slik at vanen bygges fra start. Gruppeobliger bør kreve bruk av Git-repo, ikke bare anbefale det. Commits synliggjør hvem som faktisk bidrar. Det er rettferdig for den som gjør jobben, og en mild påminnelse til den som ikke gjør det.

Git er ikke for avansert. Det er ikke for spesielt interesserte. Det er et grunnleggende verktøy du kommer til å bruke resten av karrieren. Lær det nå, ikke første gang du sitter i et jobbintervju og innser at du egentlig ikke kan det. Du kan takke meg senere.
